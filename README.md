### ✅ Commit Convention

#### 1. 커밋 메시지 형식
`Type: 요약 메시지 (날짜)`

#### 2. 커밋 타입 상세
| Type | 설명 | 해당 파일 예시 |
| :--- | :--- | :--- |
| **docs** | 학습 이론 정리 및 문서 수정 | `1230_study.md`, `1231_study.md`, `README.md` |
| **feat** | 새로운 실습 코드 및 예제 파일 추가 | `kyuhyun.txt`, `kyuhyungui.txt` 등 코드/텍스트 실습 |
| **fix** | 잘못된 내용 수정 및 오타 정정 | 학습 내용 중 틀린 정보나 오타 수정 시 |
| **refactor** | 파일 위치 이동 및 폴더 구조 변경 | 폴더 경로 변경, 파일명 변경 (내용 수정 없음) |

#### 3. 실제 사용 예시
| 날짜 | 작업 내용 | 커밋 메시지 예시 |
| :--- | :--- | :--- |
| **12/30** | 프로젝트 관리 이론 정리 | `docs: 1230 프로젝트 관리 및 버전 관리 정리 (1230)` |
| **12/31** | Git CLI 명령어 실습 | `feat: 1231 Git CLI 명령어 실습 파일 추가 (1231)` |
| **공통** | 리드미 학습 현황 업데이트 | `docs: README 학습 일지 업데이트 (0101)` |
